## Overview

Browser profiles allow you to persist cookies, local storage, and authentication state. Once a profile is created, you can attach it to any session to maintain your login state.

The workflow involves:
1. **Create a Profile**: Establish a permanent store for browser data.
2. **Open an Interactive Session**: Log in manually or via the agent to save cookies to the profile.
3. **Reuse the Profile**: Pass the `profile_id` to future sessions to remain logged in.

## Example Workflow

<CodeGroup>

```python Python
from smooth import SmoothClient

client = SmoothClient(api_key="cmzr-YOUR_API_KEY")

# 1. Create a persistent profile
profile = client.create_profile(profile_id="gmail-user")

# 2. Open a session to log in
with client.session(profile_id=profile.id) as session:
    print(f"Go to this URL to log in: {session.live_url()}")
    # ... perform manual login ...
    session.run_task("Verify I am logged in")

# 3. Later, run an automated task using the same profile
with client.session(profile_id="gmail-user") as session:
    session.run_task("Read my latest email")
    print(session.result())
```

</CodeGroup>
